BEFORE YOU START
----------------
1. You need a GitHub account
2. You need to create a Personal Access Token (for authentication)


STEP 1: CREATE GITHUB PERSONAL ACCESS TOKEN
--------------------------------------------
1. Go to: https://github.com/settings/tokens
2. Click: "Generate new token" ‚Üí "Generate new token (classic)"
3. Give it a name: "GitLab MR Conflict Resolver"
4. Select scopes:
   ‚úì repo (all)
5. Click "Generate token"
6. COPY THE TOKEN - you'll need it for pushing!
   Example: ghp_1234567890abcdefghijklmnopqrstuvwxyz


STEP 2: CREATE GITHUB REPOSITORY
---------------------------------
1. Go to: https://github.com/new
2. Repository name: gitlab-mr-conflict-resolver
3. Description: "üîß GUI tool to automatically resolve GitLab merge request conflicts"
4. Choose: Public or Private
5. DO NOT initialize with README, .gitignore, or license
   (We already have these files!)
6. Click "Create repository"


STEP 3: RUN THE PUBLISH SCRIPT
-------------------------------
Option A - Use the automated script:
   1. Double-click: publish-to-github.bat
   2. Enter your GitHub username when prompted
   3. Enter your Personal Access Token when prompted for password
   4. Done!

TROUBLESHOOTING
---------------

Problem: "Permission denied (publickey)"
Solution: Use HTTPS URL, not SSH. URL should be:
         https://github.com/YOUR_USERNAME/gitlab-mr-conflict-resolver.git

Problem: "Authentication failed"
Solution: Make sure you're using Personal Access Token as password,
         NOT your GitHub password

Problem: "Repository not found"
Solution: Make sure you created the repository on GitHub first
         with exact name: gitlab-mr-conflict-resolver

Problem: "Updates were rejected"
Solution: Make sure repository is empty (no README or files)
         when you created it
